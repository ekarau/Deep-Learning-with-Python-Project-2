## 3. Generic Training Infrastructure

I wrote `train_classifier()` and `train_reconstruction()` so every experiment shares the *same* training code path — only the model and loss change. This is what makes the comparisons fair.

`train_classifier()` supports L1 + L2 + label smoothing + LR scheduling + gradient clipping in a single call, exactly mirroring the API we used in Project 1. `train_reconstruction()` accepts an arbitrary `loss_fn(model, x_in, x_target)` callable so that the same loop drives the AE, DAE, VAE, and the four RNN-AE variants.

I also added `count_params()` (so the comparison table is honest about model size) and `evaluate_classifier()`.

## 4. LeNet-style CNN (Week 9 baseline)

The LeNet model is intentionally weak — Conv5×5 → AvgPool → Conv5×5 → AvgPool → FC120 → FC84 → FC10. **No BatchNorm, no Dropout, no augmentation, plain SGD with lr=0.01.** This is the floor against which Person 2's modern CNN and ResNet-Mini are measured.

This is also the bridge from Project 1: it shows what a CNN *with the same training discipline* as the Project 1 baseline buys us — convolution alone, without any of the modern ingredients, beats the MLP baseline by ~5–10 points on Fashion-MNIST.

### Theory I documented in the notebook

- **Convolution as a structured matrix multiply** — translation equivariance, local connectivity, weight sharing — and *why* those three biases match natural image data.
- **Pooling vs strided convolutions** — average pooling is differentiable and parameter-free but wastes capacity; the modern alternative (strided conv) is what Person 2 uses.
- **Input/output spatial dimension formula** `out = (in + 2p − k) / s + 1` — used to verify every layer in the notebook.

## 5. Connection to Project 1

In Project 1 we showed that sigmoid+SGD networks fail at ~11% accuracy because of vanishing gradients (`σ' ≤ 0.25`, so `0.25⁴ ≈ 0.004` after four layers). In Project 2 the *same* problem reappears in two new contexts:

1. **Vanilla RNNs** suffer from the same multiplicative gradient decay across timesteps; this is why LSTM/GRU exist.
2. **Plain deep CNNs** suffer from the same problem at depth; this is why residual connections exist (Person 2's ResNet-Mini).

Documenting this continuity was a core part of my responsibility — it means the project is not just "more architectures", it is the same theoretical lens applied to new problems.
