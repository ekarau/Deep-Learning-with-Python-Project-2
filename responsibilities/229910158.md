## 1. ConvAE architecture

I deliberately mirrored Person 2's ModernCNN encoder style so the comparisons stay consistent. Every choice is explicit:

```
encoder:
  Conv3×3(32, stride=2) → ReLU      # 28 → 14
  Conv3×3(64, stride=2) → ReLU      # 14 → 7
  Conv3×3(64)            → ReLU
  Flatten → Linear(64·7·7 → 32)     # latent dim = 32

decoder:
  Linear(32 → 64·7·7) → reshape (B, 64, 7, 7)
  Upsample(×2, nearest) → Conv3×3(32) → ReLU       # 7  → 14
  Upsample(×2, nearest) → Conv3×3(16) → ReLU       # 14 → 28
  Conv3×3(1) → Sigmoid
```

### Key design decisions

| Decision | Choice | Why |
|---|---|---|
| Bottleneck dim | 32 | Small enough to *force* compression (the input is 784-dim), large enough that the model can still reconstruct visible structure. |
| Reconstruction loss | MSE | Pixels are continuous in [0,1]; MSE corresponds to a Gaussian observation model and is the textbook AE choice. |
| Decoder upsampling | **Upsample + Conv** | Transposed convolutions create the well-known checkerboard artefact (Odena, Dumoulin & Olah, 2016). On 28×28 grayscale that artefact is very visible. Upsample-then-Conv produces visibly smoother reconstructions at the cost of ~5% more parameters. |
| Output activation | Sigmoid | Pixels live in [0,1]; Sigmoid keeps the prediction in the same range. |
| Optimizer | Adam, lr=1e-3 | Standard default. SGD would also work but converges much slower for the given epoch budget. |

I documented the **Upsample vs ConvTranspose trade-off** as a numbered table in the notebook so it can be referenced directly from the Wk 9 checklist.

## 2. Denoising Autoencoder

Same architecture, identical hyperparameters — only the input pipeline changes. For each batch I sample `x_noisy = clamp(x + 0.3 · ε, 0, 1)` with `ε ~ N(0, I)` and train the model to map `x_noisy → x`. This is implemented with the `noise_fn` parameter I added to `train_reconstruction()`, so the train-loop code is shared with the vanilla AE.

### Why σ = 0.3
- σ = 0.1 — almost imperceptible noise; the DAE collapses to the vanilla AE.
- σ = 0.3 — visibly corrupted but reconstruction still feasible. This is the value Vincent et al. (2008) used in their original DAE paper for MNIST-class data.
- σ = 0.5 — over-corruption: the network starts hallucinating. Useful as a future-work ablation but out of scope.

### Why train DAE at all?

The DAE is *not* about reaching a lower MSE on clean data — its MSE is in fact slightly *worse* than the vanilla AE on clean inputs. The point is that the DAE's representations are demonstrably **more robust to perturbation**, which is exactly the property you want for downstream tasks (anomaly detection, transfer learning) and for the conceptual bridge to **score-based / diffusion models** in Person 5's bonus section.

## 4. Latent-space t-SNE

I wrote `latent_tsne(model, loader, …)` — encodes a sub-sample of test images, runs `sklearn.manifold.TSNE(perplexity=30, init='pca')`, and scatter-plots the 2D projection coloured by class label.

For the vanilla AE (`09_ae_tsne.png`) the classes form **partially-separated clusters even though the AE was never given any label information**. This is the textbook empirical demonstration that reconstruction is enough of a signal for the encoder to discover semantic structure — the same observation that motivates self-supervised pre-training.

The same helper is reused on the VAE's posterior μ in Person 4's section, which makes the two figures directly comparable. The VAE clusters are tighter and better-separated, which is the qualitative payoff of the KL regulariser.
